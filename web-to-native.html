Web to Native（Web から Native の呼び出し）¶

URLフォーマット¶

Web からネイティブのメソッドを呼び出す際の URL のフォーマット

メソッド呼び出しが1つの場合¶

例：connectDendama("ABCDEF012345", "電玉1")

api-dendama://connectDendama/ABCDEF012345/%E9%9B%BB%E7%8E%891

メソッド呼び出しが複数の場合¶

  • /-/ でメソッドをつなげる。
  • 引数が "-" の場合、その引数をURLエンコードすれば問題ない。

例：connectDendama("ABCDEF012345", "電玉1") と writeDendama("電玉1")

api-dendama://connectDendama/ABCDEF012345/%E9%9B%BB%E7%8E%891/-/writeDendama/%E9%9B%BB%E7%8E%891

同時に呼び出しができないメソッド¶

  • connectDendama
  • disconnectDendama
  • executeLED
  • lightLED
  • readBattery
  • readFirmwareRevision
  • removeNotify
  • scanDendama
  • setNotify
  • stopScanDendama
  • requestProducts
  • requestPayment
  • finishPayment
  • restorePayments

BLEが関係するメソッドは同時に呼び出すと機能しない場合がある。

ただし、上記のBLEが関係するメソッドと、上記以外のメソッドは同時に呼び出してもよい。

現状のWebでの電玉接続時挙動¶

  • 電玉接続時は下記のフローでwebからnativeの関数を呼び出している
  1. 電玉の検索
    • api-dendama://scanDendam
  2. 電玉の検出
    • discover_dendama
  3. 電玉の接続
    • api-dendama://connectDendama
  4. 電玉の接続成功 (接続失敗時はメッセージ表示でユーザにリトライを依頼)
    • success_connect
  5. HardwareRevisionの読み込み+オフラインの猶予時間の設定(5秒)
    • api-dendama://readHardwareRevision/XXX/-/parameter/offlineExtension/5
  6. HardwareRevisionの取得
    • hardware_revision
  7. FirmwareRevisionの読み込み
    • api-dendama://readFirmwareRevision/
  8. FirmwareRevisionの取得
    • firmware_revision
  9. 電源情報のNotify設定
    • api-dendama://setNotify/[dendama_uuid]/battery
  10. 電源情報のNotify設定成功通知
    • success_update_notify
  11. 技情報のNotify設定(受け取った時のページによってはrequestSensorValuesも同時呼び出し)
    • api-dendama://setNotify/[dendama_uuid]/sensor
  12. 技情報のNotify設定成功通知
    • success_update_notify
  13. 紫点灯(0.2秒)
    • api-dendama://lightLED/
  14. 電源情報の読み込み（Notifyの初回が来るのが少し時間がかかるため）
    • api-dendama://readBattery/

executeLED( deviceId, color, location, term, method ) （ 非推奨：2.0〜 ）¶

LEDの稼働

  • deviceId 電玉のデバイスID
  • color 光らせるLED
    • red
    • green
  • location 光らせる皿
    • big
    • small
    • base
    • spike
  • term 光らせる秒数
    • 0: 消灯
    • 1〜255: 点灯時間（例：1 → 0.1秒、10 → 1秒、255 → 25.5秒）
  • method 点灯方法
    • 0: 点灯
    • 1〜10: 点滅（例：1 → 0.1秒ごとに点滅、5 → 0.5秒ごとに点滅 10 → 1秒ごとに点滅）

lightLED( deviceId, location, color, volume, up_time, stay_time, down_time ) （2.0〜）¶

LEDの稼働

  • location 光らせる皿
    • big
    • small
    • base
    • もしくは、上記の組み合わせ
  • color 光らせる色
    • blue
    • green
    • cyan
    • red
    • purple
    • yellow
  • volume 光量
    • 0: 消灯
    • 1: 小
    • 2: 中
    • 3: 大
  • up_time 立ち上げ時間
    • 0: 即時点灯
    • 1〜255: 立ち上げ時間（例：1 → 0.1秒、10 → 1秒、255 → 25.5秒）
  • stay_time 点灯時間
    • 0: 即時立ち下げ
    • 1〜255: 点灯時間（例：1 → 0.1秒、10 → 1秒、255 → 25.5秒）
  • down_time 立ち下げ時間
    • 0: 即時消灯
    • 1〜255: 立ち下げ時間（例：1 → 0.1秒、10 → 1秒、255 → 25.5秒）
