How to connect with AWS dev/prd servers and commands

Step 1:

Get the credentials for the server and the .pem file

hostname (ip)

username

port

Then using the ssh connection connect to the server:

// I use a conf file for manage all the connection
ssh -i file.pem username@ip-address
ssh dev-dendama

// connection, check the ip of the server to make sure you are in the prd/dev

The ip 000000 is the dev server only for develop

ubuntu@ip-000000:~/dendama-webapp$
~/dendama-webapp$ cd dendama-webapp

// you can check the content of the root with ls command

ubuntu@ip-0000000:~/dendama-webapp$ ls

Dockerfile README.md dendama docker-compose.yml groups.json manage.py nginx requirements

// list of commands that you can use

// to stop the server
sudo docker-compose down

// to rebuild the server content, this is used when you do any modification on the database models or configuration of docker/python

sudo docker-compose build

// restart the server

sudo docker-compose up -d

// this will delete any container that was created to get free space in the disc

sudo docker system prune
